License

I. Definitions

Source code - the decompiled code of a project

Software - a compiled binary version of a project

Open source - software and source code that can be freely distributed and or modified

Firm source - software and source code that can be modified, redistributed, with conditions. Source code may not be available

Closed source - software and source code that cannot be modified or redistributed

For-profit - A program that requires purchase of any sort to use part or whole of itself.

II. Expectations

Software written by the author (hereby "Me", "I", "My" or "Brennan") cannot be distributed, modified, decompiled, or in any way changed for anything other than personal or educational use regardless of status of source code. All my projects may not be used in a for-profit manner, nor may any forks of any of my projects.

III. Licenses

All my code must fall into 3 categories. Open source, firm source or closed source. In order to identify these categories, I tag all my projects in a certain manner.

All rights reserved / Do not distribute - Firm source or Closed Source. If sourcecode is made available for these projects, than assume the terms listed below for firm-source projects. Otherwise, assume that they are closed-source projects.

Freeware - Firm source. See below terms for how these projects are to be treated.

Open source - Open source. These projects are unconditional and can be edited in any way you feel like, but must follow the unconditional terms listed below.

IV. Unconditional Terms

All my projects must be copyrighted under their original licenses. By changing the license of a project that you modify you discredit the author of the orignal project.

All my projects that are changed must include a link to the original source, and must clearly state that they are originally written by me - regardless of the amount of code that is modified.

Every section of code that is modified MUST clearly state that it has been modified, and by whom it was modified.

V. Firm Source Terms

Firm Source must follow the Unconditional Terms as stated in section IV of this license. Code that qualifies as Firm Source may not be distributed without permission from the original author. Firm source code may not be redistributed for-profit.

VI. Closed Source Terms

Closed source may not be redistributed, modified, decompiled or changed for any reason, including but not limited to educational purposes, profitable use, or personal use.

VII. Forks, Clones, and Additional Licenses.

Forks and clones of my projects regardless of the license may also be copyrighted under seperate licenses, but this can only be done conditionally. Additional copyrights may only protect modified or additional sections of source code that differ from the original source. Clones of clones and forks of forks must continue to be copyrighted in this manner. Even if no original code remains, the original author MUST be listed in the credits, with what they did to start the project, and a link to the original project must be included without question.

VIII. Breaking the Rules

Any code found breaking these rules will be issued a warning to change the program. If they refuse to comply a DMCA-Takedown notice will be sent to the authors.
